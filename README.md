# geolog-dashboard

鑽探報告判讀儀表板 · Borehole Report Dashboard

把一份鑽探報告的逐孔資料,整理成一頁式地質儀表板:簡化地層比較圖、逐孔柱狀圖(含 N 值曲線)、地下水位分析、開挖穩定初判(含計算式),以及可當現場監督檢查表用的重點摘要。

純前端單檔,放上 GitHub Pages 即可分享給同事,**不需要後端、不上傳任何資料**。每個案件的資料以一個 `.json` 檔在本機進出與互傳。

## 🔒 保密原則(重要)

這是**公開**的靜態網站——任何寫進 `index.html` 或提交到 repo 的內容,所有開啟連結的人都看得到。因此:

- 網站**預設開成空白**,他人開啟連結看不到任何案件資料。
- 內建的「載入虛構示範」只含**完全虛構、與任何實際基地無關**的練習資料(`sample.json`),可安全公開。
- **真實案件資料絕不可寫入程式或提交到 repo**,只當私人 `.json` 檔保管,需要時在本機「匯入 JSON」使用。
- 資料僅存在使用者當下的瀏覽器,不上傳、不留存於網站。

## 功能

| 區塊 | 內容 |
|------|------|
| ① 簡化地層比較圖 | 全部鑽孔並排於同一深度尺;開挖底面、壁趾以橫貫參考線拉過所有孔。**各孔獨立呈現,不做跨孔內插**,避免假性連續剖面。 |
| ② 柱狀圖(逐孔詳圖) | 選孔檢視,疊 SPT-N 值曲線、標註取樣方式(S 劈管 / T 薄管)與 Su / PI / φ;卵礫石 N>100 標記假性高 N 值。 |
| ③ 地下水位分析 | 比對觀測井(O.W.)與水壓計(P.Z.)水頭,水頭差自動研判隔水層;計算開挖底面上舉力與三層級設計水位。 |
| ④ 檢核點(含計算式) | 依底面/壁趾土層自動點亮破壞模式,即時計算 FS:隆起、砂湧/管湧、上舉(受壓水)、水浮力。每張卡片列出公式、代入值、及格線與**前提假設**。 |
| ⑤ 重點摘要 | 自動歸納最弱層、主控破壞模式、地下水威脅,並產生前提條件檢查清單。 |

## 線上使用

1. 開啟網站,預設為**空白**(不顯示任何案件)。
2. **匯入 JSON** 載入你保管的案件檔,或 **編輯資料 / 新增鑽孔** 從頭輸入。需要看工具怎麼運作時,可按 **載入虛構示範**。
3. 編輯抽屜中逐孔逐層填寫,或在「JSON 直接編輯」貼上整份資料。
4. 設計參數(開挖深度、連續壁貫入 D、設計水位、結構自重等)填一次,④ 的 FS 會即時重算。
5. **匯出 JSON** 將此案件存成檔案(自己保管,勿提交到公開 repo);同事用 **匯入 JSON** 即可還原同一份儀表板。

---

## 部署到 GitHub Pages

1. 建立 repo `geolog-dashboard`,把 `index.html`(與 `README.md`、`sample.json`)放進根目錄。
2. repo → **Settings → Pages**。
3. **Source** 選 `Deploy from a branch`,分支選 `main`、資料夾選 `/ (root)`,儲存。
4. 幾分鐘後取得網址 `https://<你的帳號>.github.io/geolog-dashboard/`,分享給同事即可。

整個網站就是單一 `index.html`,無建置步驟。

## 資料格式(JSON)

可在介面內編輯,也可用任何編輯器手改。最小結構如下(完整範例見 `sample.json`):

```json
{
  "name": "案件名稱",
  "loc": "基地位置",
  "design": {
    "De": 20.35,        // 開挖深度 (m)
    "D": 20.0,          // 連續壁貫入深度 (m)
    "q": 2,             // 地表超載 (t/m²)
    "w_hi": 1.0,        // 設計高水位 GL- (m)
    "w_no": 2.0,        // 常時水位 GL- (m)
    "w_lo": 4.0,        // 低水位 GL- (m)
    "w_struct": 31,     // 結構自重 (t/m²)
    "Df": 19.7,         // 基礎底深 (m)
    "confHead": null,   // 承壓水頭 GL- (m),選填;填了才啟動上舉檢核
    "Nc": 5.7           // 隆起承載因數
  },
  "boreholes": [
    {
      "id": "BH-1",
      "total": 60,                       // 孔深 (m)
      "ow": 2.0,                         // 觀測井水位 GL- (m)
      "pz": { "depth": 47, "head": 6.0 },// 水壓計埋深與水頭 GL- (m);無則填 null
      "layers": [
        { "t": 0.9, "b": 5.4, "uscs": "CL", "name": "粉土質黏土",
          "N": 3, "LL": 40, "PI": 20, "Su": 1.8, "phi": null,
          "gamma": 1.8, "sample": "T" }
        // t=頂深, b=底深 (m);Su (t/m²);phi 摩擦角(°);gamma 單位重 (t/m³);sample: "S"|"T"
      ]
    }
  ]
}
```

**填寫原則(沿用逐孔判讀流程):**

- 依各鑽孔柱狀圖**各自獨立記載**的層位填寫,**不要直接套用簡化土層表(如表 3.1)的平均值**;簡化表僅作檢核。
- 分辨「卵礫石層(GW/GP/GM)」本身與「岩層中夾礫石顆粒」的描述差異,避免誤判層位深度與厚度。
- 雙重符號(如 `SM-CL`)直接照填,系統會以斜線圖樣標示為邊界土壤。

## 計算與免責

④ 檢核點為**概略初判**,用於快速掌握風險量級與餘裕,**不取代大地技師的正式穩定分析**。重點假設:

- **隆起**:`FS = Nc·Su / (γ·De + q)`,簡化太沙基式,**假設未做地質改良**。報告若以「地改率、改良深度」為前提達 FS≥1.2,應以技師結論為準並落實現場改良範圍。
- **砂湧**:`FS = 2·γ′·D / (γw·ΔHw)`(Terzaghi),底面/壁趾為黏土時自動判 N/A(機制不成立),但仍須查連續壁接頭水密性防局部管湧。
- **上舉**:`FS = γ_cap·t_cap / (γw·h_w)`,需填承壓水頭才啟動。
- **浮力**:`FS = 結構自重 / U`,`U=(Df−水位)×γw`;及格線施工 1.03、完工 1.07。
- 未填 `Su` 的黏土層,系統以 `Su ≈ 0.6 × N`(t/m²)暫估,可被覆寫。

及格線參考:隆起/貫入 ≥ 1.2、砂湧 ≥ 1.5(部分取 2.0)、浮力 1.03(施工)/ 1.07(完工)。
