# geolog-dashboard

鑽探報告判讀儀表板 · Borehole Report Dashboard

把一份鑽探報告的逐孔資料，整理成一頁式地質儀表板：簡化地層比較圖、逐孔柱狀圖（含 N 值曲線）、地下水位分析、開挖穩定初判（含計算式），以及可當現場監督檢查表用的重點摘要。

純前端單檔，放上 GitHub Pages 即可分享給同事，**不需要後端、不上傳任何資料**。每個案件的資料以一個 `.json` 檔在本機進出與互傳。

## 🔒 保密原則（重要）

這是**公開**的靜態網站——任何寫進 `index.html` 或提交到 repo 的內容，所有開啟連結的人都看得到。因此：

- 網站**預設開成空白**，他人開啟連結看不到任何案件資料。
- 內建的「載入虛構示範」提供兩種**完全虛構、與任何實際基地無關**的情境（`sample.json`、`sample-terrace.json`），可安全公開。
- **真實案件資料絕不可寫入程式或提交到 repo**，只當私人 `.json` 檔保管，需要時在本機「匯入 JSON」使用。
- 資料僅存在使用者當下的瀏覽器，不上傳、不留存於網站。

## 內建示範情境（皆為虛構）

| 情境 | 地層特徵 | 主要觸發的檢核 |
|------|----------|----------------|
| **A · 台北盆地軟弱黏土** | 軟弱黏土為主、開挖底面落在黏土層 | **隆起**（底面黏土）；砂湧 N/A（黏土低透水） |
| **B · 砂礫質台地** | 紅土薄層 + 厚砂層 + 卵礫石、開挖底面落在砂層 | **砂湧**（底面砂層）；隆起 N/A（底面非黏土） |

兩種情境用來展示工具如何依底面土層自動點亮不同破壞模式，並各自帶出對應的計算式與安全係數。

## 功能

| 區塊 | 內容 |
|------|------|
| ① 簡化地層比較圖 | 全部鑽孔並排於同一深度尺；開挖底面、擋土壁趾以橫貫參考線拉過所有孔。**各孔獨立呈現，不做跨孔內插**，避免假性連續剖面。 |
| ② 柱狀圖（逐孔詳圖） | 選孔檢視，疊 SPT-N 值曲線、標註取樣方式（S 劈管 / T 薄管）與 Su / PI / φ；卵礫石 N>100 標記假性高 N 值。 |
| ③ 地下水位分析 | 比對觀測井（O.W.）與水壓計（P.Z.）水頭，水頭差自動研判隔水層；計算開挖底面上舉力與三層級設計水位。 |
| ④ 開挖穩定檢核（含計算式） | 依底面/擋土壁趾土層自動點亮破壞模式，即時計算 FS：隆起、砂湧/管湧、上舉（受壓水）、水浮力。每張卡片列出公式、代入值、及格線與**前提假設**。 |
| ⑤ 重點摘要 | 自動歸納最弱層、主控破壞模式、地下水威脅，並以燈號標示前提條件檢查清單。 |

## 操作流程

### STEP 1 · 載入資料

**第一次使用（從頭輸入）**

1. 點右上角 **「編輯資料」** 開啟抽屜，或點空白頁的 **「新增鑽孔 / 編輯資料」**。
2. 在「鑽孔資料」點 **＋ 新增鑽孔**，填入孔號、孔深、觀測井水位（GL- m）。
3. 逐孔逐層填寫：頂深、底深、USCS 符號、土層名稱、N 值、Su、單位重、取樣方式（S/T）。
4. 填寫原則：依各鑽孔柱狀圖**各自獨立**記載的層位輸入，**不套用簡化土層表（如表 3.1）的平均值**。

**已有舊案（匯入 JSON）**

- 點工具列 **「匯入 JSON」**，選取你保管的 `.json` 檔，儀表板即時還原。

**初次試用（載入虛構示範）**

- 工具列下拉 **「載入虛構示範」**，選情境 A（軟弱黏土）或 B（砂礫台地），看看完整的跑完狀態。

---

### STEP 2 · 填寫設計參數

在「編輯資料」→「專案與設計參數」中填入：

| 欄位 | 說明 |
|------|------|
| 開挖深度 De (m) | 土方開挖至設計底面的深度 |
| 擋土措施深度 D (m) | 連續壁／鋼板樁等從地面到趾端的**總深度**（壁趾 = GL − D） |
| 地表超載 q (t/m²) | 開挖邊緣預期堆載（機具、材料，常取 2 t/m²） |
| 設計高水位 GL- (m) | 最不利水位，用於浮力與上舉力計算 |
| 常時水位 GL- (m) | 施工期代表水位，用於隆起 FS 計算 |
| 低水位 GL- (m) | 降水後最低水位，用於施工評估 |
| 結構自重 (t/m²) | 完工後地下室結構重量，用於抗浮計算 |
| 基礎底深 Df (m) | GL 至最深一層地下室底板底面的深度（鋪 PC 10cm 後約 De − 0.1m） |
| 承壓水頭 GL- (m) | 選填；填了才啟動「上舉（受壓水）」檢核 |

填完後，④ 開挖穩定檢核的 FS 會**即時重算**。

---

### STEP 3 · 判讀五個區塊

依序確認各區塊輸出：

1. **① 簡化地層比較圖** — 確認各孔的開挖底面（紅線）及擋土壁趾（虛線）落在哪個土層；水位標示是否合理。
2. **② 柱狀圖（逐孔詳圖）** — 切換鑽孔，逐孔核對 N 值曲線與取樣深度；注意 N>100 的卵礫石段（假性高 N 值）。
3. **③ 地下水位分析** — 確認 O.W. 與 P.Z. 水頭差是否研判出隔水層；確認開挖底面上舉力量級。
4. **④ 開挖穩定檢核** — 查各破壞模式的 FS 與及格線；PASS / 邊際 / 不足 三種結果各有對應行動。
5. **⑤ 重點摘要** — 查燈號清單（🔴紅 / 🟡黃 / 🟢綠 / ⚪灰），逐項確認地質改良、設計水位、超載管制、降水計畫等前提是否落實。

判讀完成後，點右上角 **「匯出 JSON」** 將案件資料存成 `.json` 檔自行保管，下次直接匯入即可還原。

## 部署到 GitHub Pages

最快的方式是直接 **Fork** 本 repo，不需要自己建檔或設定任何東西。

1. 登入 GitHub，開啟本 repo 頁面，點右上角 **「Fork」**，選自己的帳號，建立副本。
2. 進入你 Fork 後的 repo（`https://github.com/<你的帳號>/geolog-dashboard`）。
3. 點上方 **「Settings」** → 左側選單 **「Pages」**。
4. **Source** 選 `Deploy from a branch`，Branch 選 `main`、資料夾選 `/ (root)`，按 **Save**。
5. 等約 1–2 分鐘，頁面上方會出現網址：`https://<你的帳號>.github.io/geolog-dashboard/`。
6. 將網址分享給同事，對方開啟後看到空白儀表板，匯入各自保管的 `.json` 即可使用。

整個網站就是單一 `index.html`，Fork 後不需要任何建置步驟。

## 資料格式（JSON）

可在介面內編輯，也可用任何編輯器手改。最小結構如下（完整範例見 `sample.json`）：

```json
{
  "name": "案件名稱",
  "loc": "基地位置",
  "design": {
    "De": 20.35,        // 開挖深度 (m)
    "D": 36.0,          // 擋土措施深度，從地面到趾端的總深度 (m)
    "q": 2,             // 地表超載 (t/m²)
    "w_hi": 1.0,        // 設計高水位 GL- (m)
    "w_no": 2.0,        // 常時水位 GL- (m)
    "w_lo": 4.0,        // 低水位 GL- (m)
    "w_struct": 31,     // 結構自重 (t/m²)
    "Df": 19.7,         // 基礎底深 (m)
    "confHead": null,   // 承壓水頭 GL- (m)，選填；填了才啟動上舉檢核
    "Nc": 5.7           // 隆起承載因數
  },
  "boreholes": [
    {
      "id": "BH-1",
      "total": 60,                        // 孔深 (m)
      "ow": 2.0,                          // 觀測井水位 GL- (m)
      "pz": { "depth": 47, "head": 6.0 },// 水壓計埋深與水頭 GL- (m)；無則填 null
      "layers": [
        { "t": 0.9, "b": 5.4, "uscs": "CL", "name": "粉土質黏土",
          "N": 3, "LL": 40, "PI": 20, "Su": 1.8, "phi": null,
          "gamma": 1.8, "sample": "T" }
        // t=頂深, b=底深 (m)；Su (t/m²)；phi 摩擦角(°)；gamma 單位重 (t/m³)；sample: "S"|"T"
      ]
    }
  ]
}
```

**填寫原則（沿用逐孔判讀流程）：**

- 依各鑽孔柱狀圖**各自獨立記載**的層位填寫，**不要直接套用簡化土層表（如表 3.1）的平均值**；簡化表僅作檢核。
- 分辨「卵礫石層（GW/GP/GM）」本身與「岩層中夾礫石顆粒」的描述差異，避免誤判層位深度與厚度。
- 雙重符號（如 `SM-CL`）直接照填，系統會以斜線圖樣標示為邊界土壤。

## 計算與免責

④ 開挖穩定檢核為**概略初判**，用於快速掌握風險量級與餘裕，**不取代大地技師的正式穩定分析**。重點假設：

- **隆起**：`FS = Nc·Su / (γ·De + q)`，簡化太沙基式，**假設未做地質改良**。報告若以「地改率、改良深度」為前提達 FS≥1.2，應以技師結論為準並落實現場改良範圍。
- **砂湧**：`FS = 2·γ′·(D−De) / (γw·ΔHw)`（Terzaghi），D−De 為擋土措施開挖底面以下的嵌入段長度；底面/擋土壁趾為黏土時自動判 N/A（機制不成立），但仍須查連續壁接頭水密性防局部管湧。
- **上舉**：`FS = γ_cap·t_cap / (γw·h_w)`，需填承壓水頭才啟動。
- **浮力**：`FS = 結構自重 / U`，`U=(Df−水位)×γw`；及格線施工 1.03、完工 1.07。
- 未填 `Su` 的黏土層，系統以 `Su ≈ 0.6 × N`（t/m²）暫估，可被覆寫。

及格線參考：隆起/上舉 ≥ 1.2、砂湧 ≥ 1.5（部分取 2.0）、浮力 1.03（施工）/ 1.07（完工）。
